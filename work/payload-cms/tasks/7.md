# Task 7: Globals (Navbar, Footer, SocialChannels, SiteSettings)

## Required Skills

Перед выполнением задачи загрузи:

- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Заполнить четыре файла-заглушки Payload Globals реальными определениями из Data Models tech-spec. Каждый Global должен содержать:

1. Полное определение полей (точно по Data Models в tech-spec).
2. `afterChange` хук с `revalidatePath('/', 'layout')` — инвалидирует весь сайт (Navbar и Footer живут в root layout).
3. Начальные значения (`defaultValue` или seed-комментарии) — извлечены из существующего JSX-кода компонентов.

Эта задача не трогает сами компоненты — только создаёт Payload-определения. Интеграция компонентов с Payload Local API будет в Task 9.

**Почему важно:**  
Globals в Payload позволяют редактировать глобальный контент (слоган, соцсети, настройки сайта) через Payload Admin UI без деплоя. Хук `revalidatePath('/', 'layout')` гарантирует, что после сохранения любого Global все страницы пересоздаются в Next.js cache в течение ≤1 секунды (SLA из user-spec US-3).

**Источники данных для seed-значений:**

| Global                         | Где брать данные                                                                                                 |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------- |
| Navbar → `slogan`              | `components/Navbar.tsx` строка 48: JSX строка внутри `<div className="text-xs text-text-muted hidden sm:block">` |
| Navbar → `menuItems`           | JSX `<Link>` элементы в Desktop Nav и Mobile Menu секциях того же файла                                          |
| Footer → `socialLinks`         | JSX `<a href="...">` внутри блока "Мы в соцсетях" в `components/Footer.tsx`                                      |
| Footer → `copyrightText`       | Bottom bar в `components/Footer.tsx`                                                                             |
| SocialChannels → `url`         | JSX `href` атрибуты на `<a>` обёртках SubCard в `components/LeftSidebar.tsx`                                     |
| SocialChannels → `name`        | JSX `title` проп SubCard в `components/LeftSidebar.tsx`                                                          |
| SocialChannels → `subscribers` | `lib/config.ts` — SOCIAL_CHANNELS объект (только subscribers, имён и URL там нет!)                               |
| SiteSettings                   | Нет seed-данных — поля пустые по умолчанию                                                                       |

**Implementation hints:**

- Каждый файл экспортирует один `GlobalConfig` объект (именованный или default — согласовать с тем, что ожидает `payload.config.ts` из Task 3)
- `afterChange` хук — синхронный (не async), `revalidatePath` не возвращает Promise
- Тип хука в Payload v3: `{ afterChange: [({ doc }) => { revalidatePath('/', 'layout') }] }`
- `defaultOGImage` в SiteSettings использует тип `'upload'` с `relationTo: 'media'` — Payload отрисует media picker в Admin UI
- Seed-значения лучше оформить в виде JSDoc-комментариев над полем (не захардкоживать в `defaultValue` — это значения для сидирования БД через Payload seed скрипт, не дефолты формы)
- Globals в Payload не требуют явного `access` — по умолчанию доступны только авторизованным пользователям (OK для наших требований)

**Dependencies:**

- Task 3 должна быть выполнена: `payload.config.ts` с imports stubs и сами файлы-заглушки должны существовать
- Task 6 (Media collection) должна быть выполнена или выполняться параллельно — `siteSettings.ts` ссылается на `relationTo: 'media'`; если Media stub ещё не готов — TypeScript может не найти тип, но `npm run build` не упадёт (Payload проверяет коллекции на runtime)

**Edge cases:**

- `lib/config.ts` содержит ТОЛЬКО subscribers строки (нет name, нет url) — брать URL из JSX href в `LeftSidebar.tsx`, имена из JSX `title` проп
- Слоган в `Navbar.tsx` — это обычная JSX строка в `<div>`, не константа и не prop. Взять текст дословно без пробелов по краям
- Footer copyright — компонент рендерит год динамически (`new Date().getFullYear()`); в Global хранить только бренд-строку `"Диджитал Паб"`, год компонент добавит сам (пока компонент не интегрирован с Payload — это seed для будущей Task 9)
