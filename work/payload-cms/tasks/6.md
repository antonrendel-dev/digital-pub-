# Task 6: Articles Collection + Media Collection

Перед выполнением задачи загрузи:

- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Заполняем два стаба коллекций, созданных в Task 3:

- `payload/collections/articles.ts` — коллекция для статей, публикуемых агентами. Не заменяет MDX-статьи (они остаются в v1), а добавляет новый источник для `/articles` страницы. Агенты и администраторы могут создавать статьи; публичный `read` возвращает только опубликованные (Where-фильтр, не просто `true/false`); только администраторы могут обновлять и удалять. `afterChange` хук ревалидирует `/articles` и конкретную страницу статьи.

- `payload/collections/media.ts` — коллекция для загрузки медиафайлов через Payload built-in upload. Хранит файлы локально в `public/uploads/` (не S3 — v1). Только администраторы могут создавать/обновлять/удалять; публичный `read` без ограничений.

Обе коллекции уже импортированы в `payload.config.ts` как стабы — задача только заполнить поля, access control и хуки.

## Details

**Files:**

- `payload/collections/articles.ts` — заполнить стаб (создан в Task 3)
- `payload/collections/media.ts` — заполнить стаб (создан в Task 3)

**Dependencies:**

- Task 3 должен быть выполнен — стабы файлов и `payload.config.ts` должны существовать
- Пакет `@payloadcms/richtext-lexical` должен быть установлен (Task 3)

**Edge cases:**

- `read` для articles — НЕ использовать `() => false` или `() => true` для не-админов. Нужен именно Where-объект: `{ status: { equals: 'published' } }`. Если вернуть `false` — публичный сайт не сможет читать статьи вообще.
- `afterChange` хук должен использовать `doc.slug` (реальный slug сохранённого документа), а НЕ строку `'/articles/[slug]'` — иначе ревалидация не найдёт страницу.
- Media коллекция НЕ требует кастомных полей — Payload автоматически добавляет `filename`, `mimeType`, `filesize`, `url`, `width`, `height` через upload-конфигурацию.
- `staticDir: 'public/uploads'` — путь относительно корня проекта. `staticURL: '/uploads'` — публичный URL-префикс. Эти значения вместе обеспечивают AC: `GET /uploads/<filename>` → 200.
- `revalidatePath` импортируется из `'next/cache'` (не из next/navigation и не из других пакетов).

**Implementation hints:**

- Для articles: `import { revalidatePath } from 'next/cache'` в начале файла.
- Для articles access `read`: паттерн `req.user?.role === 'admin' ? true : { status: { equals: 'published' } }`.
- Для media: `upload: { staticDir: 'public/uploads', staticURL: '/uploads' }` — добавить как поле объекта коллекции (не в `fields`).
- Тип возвращаемого значения для `afterChange` хука: `async ({ doc }: { doc: any }) => { ... }` или использовать типы из `payload`.
- `fields: []` для media — пустой массив допустим; Payload добавит upload-поля самостоятельно.
