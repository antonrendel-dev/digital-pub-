# Task 14: Test Audit

## Required Skills

Перед выполнением задачи загрузи:

- `/skill:test-master` — [skills/test-master/SKILL.md](~/.claude/skills/test-master/SKILL.md)

## Description

Аудит тестового покрытия всех переписанных модулей. Проверяем, что unit-тесты существуют и корректно покрывают экспортируемые функции `lib/tags.ts`, `lib/posts.ts`, `lib/postUtils.ts`, скрипты миграции `scripts/migrate-tags.ts` и `scripts/migrate-posts.ts`, а также логику дедупликации в `scripts/sync-telegram.ts`.

Задача НЕ пишет код — она читает существующие тесты и код, находит пробелы, и фиксирует результаты в `decisions.md`. Если обнаружены критические пробелы в покрытии (отсутствуют тесты для ключевых путей выполнения), задача порождает `task-14-test-fixer` через отдельный spawn.

## What to do

1. **Прочитай `lib/tags.ts`** — найди все экспортируемые функции, их сигнатуры и edge cases (невалидный slug, несуществующий slug, нормальный путь).
2. **Прочитай `lib/posts.ts`** — найди все экспортируемые функции: `getPostsByTypePaginated`, `getPostBySlug`, `getPostsByType`, `getPublishedPosts`, `getPostById`, `getPostsByTag`, `getStats`. Особо — try/catch resilience path.
3. **Прочитай `lib/postUtils.ts`** — найди `getPrimaryCategorySlug`, `FeedPost` тип, вспомогательные функции.
4. **Прочитай `scripts/migrate-tags.ts`** — найди count-assertion логику и Lexical JSON конверсию.
5. **Прочитай `scripts/migrate-posts.ts`** — найди count-assertion и field mapping (Prisma camelCase → Payload snake_case).
6. **Прочитай `scripts/sync-telegram.ts`** — найди HTTP-клиентский код: `POST /api/posts`, обработку 409 (dedup skip), 201 (happy path), логику sanitizeError.
7. **Прочитай директорию `tests/`** — найди все существующие тест-файлы, сопоставь с модулями выше.
8. **Для каждого модуля** сверь фактическое покрытие с ожидаемым (см. раздел Expected Coverage ниже).
9. **Запусти `npm test`** — убедись что все тесты зелёные (ни один из существующих не упал при переписи).
10. **Зафикси результаты** в `work/payload-cms/decisions.md` по шаблону (см. раздел Reporting ниже).
11. **Если критические пробелы найдены** — создай задачу `task-14-test-fixer` с конкретным списком недостающих тестов.

## Expected Coverage

### lib/tags.ts

| Тест                                                  | Описание                                                                                                                                                                                                        |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `getTagBySlug - invalid slug returns null`            | Передать `'../evil'` или пустую строку → вернуть `null` без вызова `getPayload()` (slugSchema.safeParse отсекает)                                                                                               |
| `getTagBySlug - valid slug returns full Tag`          | Мок `payload.find` возвращает `{ docs: [{ id: 1, name: 'SMM', slug: 'smm', tagType: 'specialization', seoTitle: 'SMM', seoDescription: 'desc', seoText: '...', h1: 'SMM вакансии' }] }` → все поля не-undefined |
| `getTagsWithCounts - returns array with count fields` | Мок двух `payload.find` (теги + посты) → результат содержит `_count.posts` или эквивалентный count field                                                                                                        |

### lib/posts.ts

| Тест                                                      | Описание                                                                                                                                                                                   |
| --------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `getPostsByTypePaginated - returns correct shape`         | Мок `payload.find` возвращает `{ docs: [...], totalDocs: 42, totalPages: 3 }` → функция возвращает `{ posts, total: 42, totalPages: 3 }`                                                   |
| `getPostsByTypePaginated - FeedPost fields non-undefined` | Payload-shaped input с `tags: [{ id: 1, name: 'SMM', slug: 'smm', tagType: 'specialization' }]` → все поля `FeedPost` определены (channelUsername, telegramMessageId, tags — не undefined) |

| `getPublishedPosts - DB unavailable returns empty array`  | Мок `getPayload()` бросает Error → catch, `console.warn('[posts] DB unavailable')`, вернуть `[]`                                                                                           |

### lib/postUtils.ts

| -------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |

### scripts/migrate-tags.ts

| Тест                    | Описание                                                                                                                                             |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `count assertion`       | После миграции `count(prisma.tag)` === `count(payload.tags)` — тест проверяет что скрипт содержит явный count-assert и кидает ошибку при расхождении |
| `6-field spot check`    | Мигрированный тег имеет все 6 не-null полей: `name`, `slug`, `tagType`, `seoTitle`, `seoDescription`, `seoText`                                      |
| `Lexical JSON validity` | `seoText` сконвертирован в валидную Lexical JSON структуру: `{ root: { type: 'root', version: 1, children: [...] } }` — не пустой объект             |

### scripts/migrate-posts.ts

## Details

**Зависимости:**

- Task 10 (Agent API Route) — должна быть выполнена: sync-скрипт использует `/api/posts`, тесты проверяют mock HTTP
- Task 11 (Sync Script Migration) — должна быть выполнена: `scripts/sync-telegram.ts` переписан на Payload REST API

**Важные проверки при аудите:**

- `getTagBySlug` должен проверять slug через `slugSchema.safeParse` ДО вызова `getPayload()` — тест на невалидный slug верифицирует эту защиту
- `getPublishedPosts` (и другие функции из `lib/posts.ts`) должны иметь try/catch с `console.warn('[posts] DB unavailable')` — паттерн из patterns.md (Decision 9: graceful degradation)
- `sync-telegram.ts` тест на 409 должен использовать mocked `fetch` (не реальный HTTP) — Jest `jest.spyOn(global, 'fetch')` или `jest.mock`
- Lexical JSON validity тест должен проверять наличие `root.children` и `root.type === 'root'` — не просто `JSON.parse` без ошибки
- Тест на `sanitizeError` в sync-скрипте: убедиться что `BOT_TOKEN` из URL не попадает в `console.error` (проверить что `sanitizeError` применяется к URL ошибкам)

**Что НЕ входит в этот аудит:**

- E2E тесты Payload admin UI (нет автоматизации — ручная проверка в Task 15)
- Integration tests afterChange → revalidatePath (нельзя воспроизвести в Jest без запущенного Next.js — проверяется в Task 15 на staging)
- `lib/tag-matcher.ts` — существующие тесты проверены в рамках Task 11, регрессий быть не должно
