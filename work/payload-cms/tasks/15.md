## Description

Полный acceptance-тест интеграции Payload CMS на staging перед деплоем в продакшн. Покрывает все сценарии user-spec и критерии приёмки tech-spec: smoke routes, auth checks, API-эндпоинты, тайминг revalidation, sync dedup, целостность миграции.

### 4. Media upload и доступ

```bash
BASE="https://staging.d-pub.ru"

# Загрузить тестовое изображение (1x1 px PNG)
UPLOAD_RESULT=$(curl -s -X POST $BASE/api/media \
  -H "Authorization: users API-Key $ADMIN_API_KEY" \
  -F "file=@/tmp/test.png;type=image/png")

echo $UPLOAD_RESULT | grep -o '"filename":"[^"]*"'
FILENAME=$(echo $UPLOAD_RESULT | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('doc',{}).get('filename',''))" 2>/dev/null)

# Создать тестовый файл если нет
python3 -c "
import struct, zlib
def png_chunk(t, d): return struct.pack('>I', len(d)) + t + d + struct.pack('>I', zlib.crc32(t+d)&0xffffffff)
data = b'\x89PNG\r\n\x1a\n' + png_chunk(b'IHDR', struct.pack('>IIBBBBB',1,1,8,2,0,0,0)) + png_chunk(b'IDAT', zlib.compress(b'\x00\xff\xff\xff')) + png_chunk(b'IEND', b'')
open('/tmp/test.png','wb').write(data)
"

# Проверить доступ к загруженному файлу
if [ -n "$FILENAME" ]; then
  curl -s -o /dev/null -w "%{http_code}" $BASE/uploads/$FILENAME
fi
```

**AC из tech-spec:**

- [ ] `POST /api/media` → 201
- [ ] `GET /uploads/<filename>` → 200

### 11. Revalidation timing (afterChange hook)

Этот шаг требует доступа к Payload admin UI — выполняется агентом через API или вручную:

```bash
# Через API: обновить тег smm и измерить время до обновления страницы
TAG_ID=$(curl -s "$BASE/api/tags?where[slug][equals]=smm" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['docs'][0]['id'])" 2>/dev/null)

BEFORE=$(date +%s%3N)
# Обновить seoText тега через Payload REST API
curl -s -X PATCH "$BASE/api/tags/$TAG_ID" \
  -H "Authorization: users API-Key $ADMIN_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"seoTitle":"QA Updated Title '$(date +%s)'"}'

# Проверять страницу до обновления
for i in 1 2 3 4 5; do
  sleep 0.5
  TITLE=$(curl -s $BASE/vacancies/smm | grep -o 'QA Updated Title [0-9]*' | head -1)
  if [ -n "$TITLE" ]; then
    AFTER=$(date +%s%3N)
    echo "Revalidated in $((AFTER-BEFORE))ms (iteration $i)"
    break
  fi
done
```

**AC из tech-spec:**

- [ ] afterChange hook на Tag fires `revalidatePath` в том же request cycle
- [ ] Изменение тега → `GET /vacancies/smm` обновляется за ≤ 3 секунды
